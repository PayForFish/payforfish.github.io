这里的2个内部调用全是DispatcherServlet内部逻辑

上面的是spring所做的最后一步，往上就是服务器（tomcat）所做的事情了。因为这里的FrameworkServlet 父类是HttpServletBean，HttpServletBean父类就是Tomcat的HttpServlet类。至此，spring所对服务器过来的请求处理全部结束，接下来是上层服务器的逻辑，暂不讨论。
综上，在spring接收到web服务器的web请求时，执行的调用链是：

拿get请求举例：
-> FrameworkServlet覆盖tomcat的HttpServlet的doGet()
-> FrameworkServlet的processRequest(HttpServletRequest request, HttpServletResponse response)
-> DispatcherServlet继承FrameworkServlet执行覆写的doService(HttpServletRequest request, HttpServletResponse response)
-> DispatcherServlet的doDispatch(HttpServletRequest request, HttpServletResponse response)
-> AbstractHandlerMethodAdapter实现HandlerAdapter的handle(HttpServletRequest request, HttpServletResponse response, Object handler)
-> RequestMappingHandlerAdapter实现抽象类AbstractHandlerMethodAdapter的handleInternal(HttpServletRequest request, HttpServletResponse response, HandlerMethod handlerMethod)
-> RequestMappingHandlerAdapter的invokeHandlerMethod(HttpServletRequest request, HttpServletResponse response, HandlerMethod handlerMethod)
-> ServletInvocableHandlerMethod的invokeAndHandle(ServletWebRequest webRequest, ModelAndViewContainer mavContainer, Object… providedArgs)
-> 执行父类InvocableHandlerMethod的invokeForRequest(NativeWebRequest request, @Nullable ModelAndViewContainer mavContainer, Object… providedArgs)
-> InvocableHandlerMethod的内部调用doInvoke(Object… args)
-> spring获取调用bean反射调用业务代码

至此完整的web服务器 -> spring -> 业务代码 的调用链就结束了。
